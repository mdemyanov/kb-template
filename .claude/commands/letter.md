# Деловая переписка

Используй skill `correspondence-2` для подготовки деловых писем.

## Параметры

- `vault_name`: {{VAULT_NAME}}
- `vault_tool`: aigrep
- `people_dir`: 10_PEOPLE
- `constraints_file`: 00_CORE/identity/constraints.md

## Алгоритм

1. Определи контекст:
   - Кому пишем (профиль из базы)
   - Какой канал (email / мессенджер / документ)
   - Какая цель (информирование / запрос / решение)

2. Учти ограничения:
   - Прочитай constraints.md
   - Проверь историю коммуникаций с адресатом
   - Учти организационный контекст

3. Подготовь черновик:
   - Структура: цель → контекст → суть → действие
   - Тон: деловой, но не сухой
   - Длина: минимально необходимая

4. Предложи варианты:
   - Основной вариант
   - Более формальный (если нужно)
   - Более краткий (если нужно)

## Использование

```
/letter [описание задачи]
```

Примеры:
- `/letter написать CEO о задержке проекта`
- `/letter ответить на запрос о ресурсах от HR`
- `/letter поздравить команду с релизом`

## Форматы

### Email
```
Тема: [краткая суть]

[Обращение],

[1-2 абзаца по сути]

[Конкретный запрос / следующий шаг]

[Подпись]
```

### Мессенджер
```
[Суть в 1-3 предложениях]
[Конкретный вопрос или действие]
```

### Документ / Memo
```
# [Название]

**Дата:**
**От:**
**Кому:**
**Тема:**

## Суть

## Контекст

## Предложение

## Следующие шаги
```
