# Knowledge Base Template

Шаблон персональной базы знаний руководителя с интегрированным AI-ассистентом.

## Концепция

**AI-ассистент — это советник, не исполнитель.**

Его задача — помочь думать лучше, а не думать вместо вас. Ключевой принцип: AI должен оспаривать, а не подтверждать. Если AI всегда соглашается — он бесполезен.

### Три уровня работы AI

| Уровень | Функции |
|---------|---------|
| **Стратегический советник** | Анализ ситуаций, генерация альтернатив, критика решений |
| **Аналитик контекста** | Поиск информации, подготовка к встречам, отслеживание связей |
| **Инструмент обработки** | Документы по шаблонам, обработка транскриптов, структурирование |

## Для кого

- **CTO** — технологическая стратегия, архитектура, команды разработки
- **CPO** — продуктовая стратегия, роадмапы, исследования
- **COO** — операционные процессы, метрики, проекты
- **HR Director** — люди, компетенции, программы развития
- **Product Manager** — проекты, риски, стейкхолдеры

## Быстрый старт

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/mdemyanov/kb-template.git ~/Documents/MyKB
cd ~/Documents/MyKB
```

### 2. Персонализируйте базу

Откройте `SETUP.md` в Claude Desktop или Claude Code и следуйте инструкциям. AI проведёт вас через:
- Интервью о вашей роли и контексте
- Замену плейсхолдеров в файлах
- Установку инструментов
- Верификацию настройки

### 3. Откройте в Obsidian

```bash
# Obsidian → Open folder as vault → выберите папку с базой
```

## Требования

| Компонент | Назначение |
|-----------|-----------|
| **macOS** | Поддерживается только macOS |
| **Obsidian** | База знаний и визуализация |
| **Claude Desktop** или **Claude Code** | AI-ассистент |
| **Ollama** | Локальный сервер для embeddings |
| **aigrep** | Семантический поиск по базе |

## Структура папок

Организация по системе [Johnny Decimal](https://johnnydecimal.com/):

```
kb-template/
├── 00_CORE/                 # Ядро: идентичность, стратегия
│   ├── identity/            # Роль, ограничения
│   ├── stakeholders/        # Ключевые люди
│   └── strategy/            # Приоритеты, бэклог
│
├── 10_PEOPLE/               # Профили людей и 1-1 встречи
│
├── 20_MEETINGS/             # Встречи
│   ├── committees/          # Регулярные комитеты
│   └── sessions/            # Разовые сессии
│
├── 30_PROJECTS/             # Проекты
│   ├── active/              # Активные
│   ├── backlog/             # В очереди
│   └── archive/             # Завершённые
│
├── 40_DECISIONS/            # Решения
│   ├── adr/                 # Architecture Decision Records
│   └── journal/             # Decision Journal
│
├── 50_KNOWLEDGE/            # Знания
│   ├── methodologies/       # Методологии и фреймворки
│   └── glossary/            # Глоссарий терминов
│
├── 60_DOMAIN/               # Предметная область (адаптируется под роль)
│
├── 90_TEMPLATES/            # Шаблоны документов
│
├── 99_ARCHIVE/              # Архив
│
├── .claude/commands/        # Slash-команды для Claude
│
├── Dashboard.md             # Главная страница
├── CLAUDE.md                # Инструкции для Claude Code
├── CLAUDE_DESKTOP.md        # Системный промт для Claude Desktop
└── SETUP.md                 # Промт персонализации
```

## Плейсхолдеры

После клонирования замените плейсхолдеры на свои значения:

| Плейсхолдер | Описание | Пример |
|-------------|----------|--------|
| `{{COMPANY}}` | Название компании | Acme Corp |
| `{{ROLE}}` | Ваша должность | CTO |
| `{{VAULT_NAME}}` | Имя vault для aigrep | acme-cto |
| `{{VAULT_PATH}}` | Путь к базе | ~/Documents/Acme_CTO |
| `{{AREAS}}` | Области ответственности | технстратегия, архитектура, команды |
| `{{STAKEHOLDERS}}` | Ключевые стейкхолдеры | CEO, CPO, Tech Leads |
| `{{CONSTRAINTS}}` | "Красные линии" | единая платформа, in-house разработка |
| `{{METHODOLOGIES}}` | Используемые методологии | OKR, ADR, Agile |
